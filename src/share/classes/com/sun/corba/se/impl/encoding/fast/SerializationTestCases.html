Java Serialization test cases

Basic design is the same as copyobject tests

Functionality tests

  1. all of the copyobject tests (most of java.util is covered)
  2. readResolve/writeReplace
  3. serialPersistentFields
  4. transient fields
  5. writeObject/readObject
  6. externalizable
  7. putFields/getFields

Performance tests

These can be taken somewhat from the StandardTest

  1. stream creation
  2. read/write primitve array 10000 elements
  3. read/write array of 1000 bean-style classes, each contains 10 fields
  4. large Map (1000 entries)
  5. large complex graph (can use copyobject code)

Eventually we need to deal with type evolution. This would also be useful to support copyobject semantics between different class loaders in the sender and receiver (particularly under OSGi).
